# JUNOS Terraform Automation Framework (JTAF)

JTAF converts Junos YANG models and XML configurations into generated artifacts for:

- Terraform providers and `.tf` test files
- Ansible roles, playbooks, and `host_vars`

This repository also includes a NETCONF mock server and CI scripts that validate an end-to-end EVPN-VXLAN provisioning workflow for both Terraform and Ansible.

## What This Repository Provides

- Python CLI tools (`jtaf-*`) to generate Terraform and Ansible artifacts
- A pyang plugin (`jtaf-pyang-plugindir`) to produce JTAF JSON from YANG
- Go provider templates under `terraform_provider/`
- NETCONF mock server under `netconf_mock/`
- CI-proven integration flow examples under `.github/workflows/`

## Prerequisites

Install the following first:

- Python `3.9+`
- Go (current stable)
- Terraform (current stable)
- Git
- `pyang` CLI (installed via Python dependencies below)

Notes:

- The NETCONF mock server itself requires Python `3.10+` when run directly.
- CI runs Ansible integration on Python `3.9` and `3.x` with version-specific constraints.

## CLI Tools

### `jtaf-pyang-plugindir`

Prints the installed path to the JTAF pyang plugin directory.

```bash
jtaf-pyang-plugindir
```

### `jtaf-provider`

Generates a Terraform provider source tree from JTAF JSON and one or more XML configs.

```bash
jtaf-provider -j <json-schema-or-> -x <config1.xml> [<config2.xml> ...] -t <device-type>
```

Key flags:

- `-j, --json-schema`: JTAF JSON schema file path, or `-` for stdin
- `-x, --xml-config`: one or more XML config files (repeat/group allowed)
- `-t, --type`: device type suffix used in output folder name

Output folder pattern: `terraform-provider-junos-<device-type>/`

### `jtaf-yang2go`

One-step wrapper around `pyang` + `jtaf-provider`.

```bash
jtaf-yang2go -p <common-dir> <yang-files...> -x <config1.xml> [<config2.xml> ...] -t <device-type>
```

Key flags:

- `-p`: YANG search paths and/or `.yang` files
- `-x`: one or more XML config files
- `-t`: device type

### `jtaf-xml2tf`

Generates Terraform `.tf` resources from XML using `trimmed_schema.json`.

```bash
jtaf-xml2tf -j <trimmed_schema.json> -x <config1.xml> [<config2.xml> ...] -t <device-type> -d <output-dir> [-u <username>] [-p <password>]
```

Key flags:

- `-j, --trimmed_json`: trimmed schema generated by provider step
- `-x, --xml-config`: one or more XML files
- `-t, --type`: device type
- `-d, --directory`: output directory for `.tf` files and `providers.tf`
- `-u, --username`: optional username for generated provider blocks
- `-p, --password`: optional password for generated provider blocks

### `jtaf-ansible`

Generates an Ansible role + playbook from JTAF JSON + XML.

```bash
jtaf-ansible -j <json-schema-or-> -x <config1.xml> [<config2.xml> ...] -t <device-type>
```

Output folder pattern: `ansible-provider-junos-<device-type>/`

### `jtaf-yang2ansible`

One-step wrapper around `pyang` + `jtaf-ansible`.

```bash
jtaf-yang2ansible -p <common-dir> <yang-files...> -x <config1.xml> [<config2.xml> ...] -t <device-type>
```

### `jtaf-xml2yaml`

Generates Ansible `host_vars/*.yaml` and a `hosts` inventory list from XML.

```bash
jtaf-xml2yaml -j <trimmed_schema.json> -x <config1.xml> [<config2.xml> ...] -d <output-dir>
```

## Step-by-Step: Provision Complete EVPN-VXLAN Example (Terraform + Ansible via NETCONF Mock)

This section mirrors the behavior validated in:

- `.github/workflows/go-terraform-provider.yml`
- `.github/workflows/ansible-provider.yml`

Run all commands from repository root unless noted.

### 1) Prepare Environment

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
pip install -r .github/requirements/ansible-test-requirements.txt
pip install .
```

Install/verify external tools:

```bash
go version
terraform version
ansible-playbook --version
```

### 2) Generate Terraform Provider (EVPN-VXLAN)

```bash
rm -rf terraform-provider-junos-vqfx-evpn-vxlan

jtaf-yang2go \
  -p examples/yang/18.2/18.2R3/common \
  examples/yang/18.2/18.2R3/junos-qfx/conf/*.yang \
  -x \
  examples/evpn-vxlan-dc/dc1/dc1-borderleaf1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-borderleaf2.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf2.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf3.xml \
  examples/evpn-vxlan-dc/dc1/dc1-spine1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-spine2.xml \
  examples/evpn-vxlan-dc/dc2/dc2-spine1.xml \
  examples/evpn-vxlan-dc/dc2/dc2-spine2.xml \
  -t vqfx-evpn-vxlan
```

### 3) Build and Install Generated Provider

```bash
cd terraform-provider-junos-vqfx-evpn-vxlan
go build ./...
go test ./...
go install .
cd ..
```

Configure Terraform dev override in `~/.terraformrc`:

```hcl
provider_installation {
  dev_overrides {
    "hashicorp/junos-vqfx-evpn-vxlan" = "$HOME/go/bin"
  }
  direct {}
}
```

Export for current shell:

```bash
export TF_CLI_CONFIG_FILE="$HOME/.terraformrc"
export TF_IN_AUTOMATION=1
```

### 4) Generate EVPN-VXLAN Terraform Files

```bash
rm -rf ci-evpn-vxlan-tf

jtaf-xml2tf \
  -j terraform-provider-junos-vqfx-evpn-vxlan/trimmed_schema.json \
  -x examples/evpn-vxlan-dc/dc1/*{spine,leaf}*.xml examples/evpn-vxlan-dc/dc2/*spine*.xml \
  -t vqfx-evpn-vxlan \
  -d ci-evpn-vxlan-tf \
  -u ci-user \
  -p ci-password
```

Rewrite generated providers to use deterministic mock ports and create devices file:

```bash
python .github/scripts/rewrite_mock_providers.py \
  --providers-file ci-evpn-vxlan-tf/providers.tf \
  --devices-file ci-evpn-vxlan-tf/mock-devices.txt \
  --bind-host 127.0.0.1 \
  --base-port 8301
```

### 5) Start NETCONF Mock for Terraform Run

```bash
python netconf_mock/netconf_mock_server.py \
  --host 127.0.0.1 \
  --username ci-user \
  --password ci-password \
  --devices-file ci-evpn-vxlan-tf/mock-devices.txt \
  --state-dump netconf-mock-state.json \
  --log-level INFO > netconf-mock.log 2>&1 &

echo $! > .netconf_mock.pid
```

### 6) Execute Terraform Provision/Reconcile/Negative Cycle

```bash
cd ci-evpn-vxlan-tf

terraform plan -refresh=false -no-color -input=false -out=tfplan-initial
terraform apply -no-color -input=false -auto-approve -parallelism=1 tfplan-initial

terraform plan -refresh=false -no-color -input=false -detailed-exitcode
# expected exit code: 0 (no-op)

python ../.github/scripts/mutate_first_ipv4_cidr.py \
  --directory . \
  --replacement 203.0.113.10/32 \
  --exclude providers.tf

terraform plan -refresh=false -no-color -input=false -detailed-exitcode -out=tfplan-updated
# expected exit code: 2 (changes)

terraform apply -no-color -input=false -auto-approve -parallelism=1 tfplan-updated
terraform plan -refresh=false -no-color -input=false -detailed-exitcode
# expected exit code: 0 (no-op)

cd ..
```

Optional drift simulation and reconcile (same CI behavior):

```bash
python .github/scripts/apply_netconf_drift.py \
  --devices-file ci-evpn-vxlan-tf/mock-devices.txt \
  --target-host "$(head -n 1 ci-evpn-vxlan-tf/mock-devices.txt | cut -d: -f1)" \
  --target-group base-config \
  --drift-ip 203.0.113.250/32 \
  --username ci-user \
  --password ci-password \
  --connect-host 127.0.0.1

cd ci-evpn-vxlan-tf
terraform plan -no-color -input=false -detailed-exitcode -out=tfplan-drift
# expected exit code: 2
terraform apply -no-color -input=false -auto-approve -parallelism=1 tfplan-drift
terraform plan -no-color -input=false -detailed-exitcode
# expected exit code: 0
cd ..
```

### 7) Generate Ansible Role/Playbook + host_vars

```bash
rm -rf ansible-provider-junos-vqfx-evpn-vxlan ci-evpn-vxlan-ansible

jtaf-yang2ansible \
  -p examples/yang/18.2/18.2R3/common \
  examples/yang/18.2/18.2R3/junos-qfx/conf/*.yang \
  -x \
  examples/evpn-vxlan-dc/dc1/dc1-borderleaf1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-borderleaf2.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf2.xml \
  examples/evpn-vxlan-dc/dc1/dc1-leaf3.xml \
  examples/evpn-vxlan-dc/dc1/dc1-spine1.xml \
  examples/evpn-vxlan-dc/dc1/dc1-spine2.xml \
  examples/evpn-vxlan-dc/dc2/dc2-spine1.xml \
  examples/evpn-vxlan-dc/dc2/dc2-spine2.xml \
  -t vqfx-evpn-vxlan

jtaf-xml2yaml \
  -x examples/evpn-vxlan-dc/dc1/*{spine,leaf}*.xml examples/evpn-vxlan-dc/dc2/*spine*.xml \
  -j ansible-provider-junos-vqfx-evpn-vxlan/trimmed_schema.json \
  -d ci-evpn-vxlan-ansible

rm -rf ansible-provider-junos-vqfx-evpn-vxlan/host_vars
cp -R ci-evpn-vxlan-ansible/host_vars ansible-provider-junos-vqfx-evpn-vxlan/
```

Prepare mock inventory and CI-style netconf playbook:

```bash
python .github/scripts/prepare_ansible_mock_inventory.py \
  --hosts-file ci-evpn-vxlan-ansible/hosts \
  --inventory-file ansible-provider-junos-vqfx-evpn-vxlan/mock-inventory.ini \
  --devices-file ansible-provider-junos-vqfx-evpn-vxlan/mock-devices.txt \
  --bind-host 127.0.0.1 \
  --base-port 8301

python .github/scripts/ansible_workflow/render_ci_netconf_playbook.py \
  --roles-root ansible-provider-junos-vqfx-evpn-vxlan/roles \
  --template-path .github/ansible/ci-netconf-playbook.yml.j2 \
  --output-path ansible-provider-junos-vqfx-evpn-vxlan/ci-netconf-playbook.yml
```

### 8) Restart NETCONF Mock for Ansible Run

Stop previous mock if still running:

```bash
if [ -f .netconf_mock.pid ]; then
  kill "$(cat .netconf_mock.pid)" 2>/dev/null || true
  rm -f .netconf_mock.pid
fi
```

Start mock with Ansible device map:

```bash
python netconf_mock/netconf_mock_server.py \
  --host 127.0.0.1 \
  --username ci-user \
  --password ci-password \
  --devices-file ansible-provider-junos-vqfx-evpn-vxlan/mock-devices.txt \
  --state-dump netconf-mock-state.json \
  --log-level INFO > netconf-mock.log 2>&1 &

echo $! > .netconf_mock.pid
```

### 9) Execute Ansible Provision/Reconcile/Negative Cycle

```bash
cd ansible-provider-junos-vqfx-evpn-vxlan

export ANSIBLE_HOST_KEY_CHECKING=False
export ANSIBLE_DEPRECATION_WARNINGS=False
export NETCONF_USERNAME=ci-user
export NETCONF_PASSWORD=ci-password

ansible-playbook -i mock-inventory.ini ci-netconf-playbook.yml
ansible-playbook -i mock-inventory.ini ci-netconf-playbook.yml

python ../.github/scripts/mutate_first_yaml_ipv4_cidr.py \
  --directory host_vars \
  --replacement 203.0.113.10/32

ansible-playbook -i mock-inventory.ini ci-netconf-playbook.yml

python ../.github/scripts/apply_netconf_drift.py \
  --devices-file mock-devices.txt \
  --target-host "$(ls -1 host_vars/*.y*ml | sort | head -n 1 | xargs -r -n1 basename | sed -E 's/\.(yaml|yml)$//')" \
  --target-group "$(python ../.github/scripts/ansible_workflow/derive_target_group.py --config-xml configs/$(ls -1 host_vars/*.y*ml | sort | head -n 1 | xargs -r -n1 basename | sed -E 's/\.(yaml|yml)$//').xml)" \
  --drift-ip 203.0.113.250/32 \
  --username ci-user \
  --password ci-password \
  --connect-host 127.0.0.1

ansible-playbook -i mock-inventory.ini ci-netconf-playbook.yml

NETCONF_PASSWORD=wrong-password NETCONF_EXPECT_FAILURE=1 \
ansible-playbook -i mock-inventory.ini ci-netconf-playbook.yml
# expected: failure

cd ..
```

### 10) Stop Mock and Validate State

```bash
if [ -f .netconf_mock.pid ]; then
  kill "$(cat .netconf_mock.pid)" 2>/dev/null || true
  wait "$(cat .netconf_mock.pid)" 2>/dev/null || true
fi

python .github/scripts/assert_netconf_mock_state.py \
  --state-dump netconf-mock-state.json \
  --require-min-commits 3
```

At this point you have completed the EVPN-VXLAN provisioning example across both generated Terraform and generated Ansible flows using a stateful NETCONF mock backend.

## Troubleshooting

- `pyang failed or produced no output`: check `-p` search paths and YANG file globs.
- Missing Terraform provider at plan/apply: verify `~/.terraformrc` `dev_overrides` path points to `$HOME/go/bin` and provider was installed with `go install .`.
- NETCONF auth failures: ensure generated provider/Ansible credentials match mock `--username` and `--password`.
- Ansible collection/module issues: confirm `ansible-core` and related dependencies were installed from `.github/requirements/ansible-test-requirements.txt`.
- Unexpected reconciliation behavior: inspect `netconf-mock.log` and `netconf-mock-state.json`.
