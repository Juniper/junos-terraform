# Terraform Provider

Go implementation of a Terraform Plugin Framework provider for Junos configuration management. Lives at `terraform_provider/`.

## Architecture

```
Terraform CLI
    ↓
Provider (main.go → provider.go)
    ↓
Config Resource (resource_config_provider.go)
    ↓
NETCONF Client (netconf/client.go)
    ↓
Junos Device (SSH:830)
```

## Core Files

| File | Purpose |
|------|---------|
| `main.go` | Provider entry point, starts plugin server |
| `provider.go` | Provider struct, schema (host/port/username/password/sshkey), Configure method |
| `config.go` | `Config{}` struct, `Client()` factory for NETCONF connections |
| `resource_config_provider.go` | Config resource with CRUD operations (template-generated) |
| `resource_file.go` | File-based resource for testing/utility |

## NETCONF Transport — `terraform_provider/netconf/`

| File | Purpose |
|------|---------|
| `client.go` | NETCONF-over-SSH client with RPC lifecycle |
| `common.go` | `Client` interface definition |

### Supported RPCs

- `<load-configuration>` — loads config into candidate datastore
- `<edit-config>` — applies patches (create/replace/delete operations)
- `<commit/>` — commits candidate to running
- `<discard-changes/>` — restores candidate from running
- `<get-configuration>` — retrieves running config

## Provider Schema

```hcl
provider "junos-{device-type}" {
  host     = "10.54.21.174"     # Device IP or hostname
  port     = 830                # NETCONF port
  username = "root"             # SSH username
  password = "secret"           # SSH password
  sshkey   = "~/.ssh/id_rsa"   # SSH key (alternative to password)
}
```

## Resource Lifecycle (CRUD)

| Operation | What Happens |
|-----------|--------------|
| **Create** | Terraform plan → XML → `load-configuration` → `commit` |
| **Read** | `get-configuration` → parse XML → update state |
| **Update** | Compute diff (patch engine) → `edit-config` with operations → `commit` |
| **Delete** | Build delete XML → `edit-config operation="delete"` → `commit` |

## Code Generation

The provider Go code is generated by Jinja2 templates from YANG-derived JSON schema:

| Template | Generates |
|----------|-----------|
| `junosterraform/templates/provider.go.j2` | Provider struct, schema, Configure |
| `junosterraform/templates/resource_config_provider.go.j2` | Resource structs, CRUD methods |
| `junosterraform/templates/config.go.j2` | Config struct, Client factory |
| `junosterraform/templates/go.mod.j2` | Go module definition |

## Go Dependencies

- `terraform-plugin-framework v1.18.0` — Terraform plugin SDK
- `nemith.io/netconf v0.0.4` — NETCONF protocol library
- `golang.org/x/crypto` — SSH support
