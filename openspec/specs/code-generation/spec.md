# Code Generation Templates

Jinja2 templates that render YANG-derived JSON schema into Go Terraform provider source code and Ansible artifacts. Lives at `junosterraform/templates/`.

## Templates

| Template | Generates | Output Location |
|----------|-----------|-----------------|
| `resource_config_provider.go.j2` | Go resource structs with CRUD methods | `terraform-provider-junos-{type}/resource_config_provider.go` |
| `provider.go.j2` | Provider struct, schema, Configure method | `terraform-provider-junos-{type}/provider.go` |
| `config.go.j2` | Config struct, Client() factory | `terraform-provider-junos-{type}/config.go` |
| `go.mod.j2` | Go module definition | `terraform-provider-junos-{type}/go.mod` |
| `ansible.j2` | Ansible task templates | `ansible-provider-junos-{type}/` |

## Template Variables

All templates receive a `data` object containing:

| Variable | Type | Description |
|----------|------|-------------|
| `data.device_type` | string | Device type suffix (e.g., "vmx", "qfx5100", "vqfx-evpn-vxlan") |
| `data['root']` | dict | Root of the filtered YANG schema tree |
| `data['root']['children']` | list | Top-level configuration nodes |

Each node in the schema tree contains:
- `name` — Element name
- `type` — YANG type or structural type
- `key` — List key fields
- `children` — Nested nodes
- `description` — YANG description

## Generation Flow

```
Filtered JSON schema (from jtaf_common.filter_json_using_xml)
    ↓
Jinja2 template engine
    ↓
For each template:
  - Render with data context
  - Write to terraform-provider-junos-{type}/ directory
    ↓
Result: Complete, buildable Go provider
```

## What Gets Generated

For a device type like `vmx-4-topo`, the output directory contains:

```
terraform-provider-junos-vmx-4-topo/
├── main.go                      ← from main.go template (provider entry point)
├── provider.go                  ← from provider.go.j2 (provider definition)
├── config.go                    ← from config.go.j2 (NETCONF config)
├── resource_config_provider.go  ← from resource_config_provider.go.j2 (resources)
├── go.mod                       ← from go.mod.j2 (module definition)
└── go.sum                       ← generated by go mod tidy
```

## Resource Code Generation

The `resource_config_provider.go.j2` template generates:
- One Go struct per YANG container/list in the filtered schema
- Terraform schema definition mapping YANG types to Terraform types
- Create method: XML builder from Terraform state → `load-configuration`
- Read method: `get-configuration` → parse XML → update Terraform state
- Update method: Diff via patch engine → `edit-config`
- Delete method: Build delete XML → `edit-config operation="delete"`

## Conventions

- Generated file names follow Go conventions (`snake_case.go`)
- Provider registry address: `tf-registry.click/juniper/jtaf-{type}`
- Resources named: `terraform-provider-junos-{type}`
- YANG namespaces stripped in resource schemas
